📚 Course Material Summary

WCAG 2.1 / 2.2

4 principles: Perceivable, Operable, Understandable, Robust (POUR). Conformance levels: A (must), AA (should — legal baseline in EU/US), AAA (could).

Keyboard Navigation

All interactive elements reachable via Tab. Visible focus indicators. Logical tab order. No keyboard traps. Skip-to-content links for repetitive nav.

Screen Readers

Semantic HTML > ARIA. Headings hierarchy (h1 → h6). Alt text for images. Form labels. Landmark roles (header/main/nav/footer).

Color Contrast

Text vs background: AA = 4.5:1 (normal), 3:1 (large 18pt+). UI components: 3:1. Don't convey info by color alone.

Automated Tools

axe-core (Playwright/Cypress integration), Lighthouse, WAVE, IBM Equal Access. Catch ~30-40% of issues. Manual testing required for the rest.

Testing Approach

Combine: automated scan (axe) + keyboard-only walkthrough + screen reader test (VoiceOver/NVDA) + zoom 200% + color blindness simulation.

📝 Homework Assignment

Accessibility Testing — Hands On

Pick a real website. Run accessibility tests covering at least three WCAG principles. Report findings with severity, WCAG criterion mapped, and remediation suggestion. Bonus: write an automated axe-core test (Playwright).

Required Tasks:

  1. Run an automated scan (axe DevTools, WAVE, or Lighthouse) and document the findings
  2. Perform a keyboard-only walkthrough of a critical user flow
  3. Check color contrast on key text/UI elements
  4. Map each issue to a WCAG 2.1 success criterion (e.g. 1.4.3, 2.1.1, 4.1.2)
  5. Suggest remediation for each issue

📊 Evaluation Rubric

Criteria Weight What we look for
Issue Identification25%Variety of issue types (perceivable, operable, understandable, robust). Not just one category.
WCAG Mapping20%Each finding mapped to a specific WCAG 2.1 success criterion (e.g. 1.4.3 Contrast Minimum).
Severity / Impact15%Reasoned severity (Blocker / Critical / Major / Minor) tied to user impact, not just tool output.
Remediation Suggestions20%Concrete, actionable fixes. Not just "add ARIA". Mention semantic HTML alternatives where applicable.
Methodology10%Combines automated + manual (keyboard, screen reader, zoom). Doesn't rely solely on one tool.
Presentation & Structure10%Clean formatting, professional layout, easy to follow.
